# 03. Блок «Применение» (ключевой вау-элемент)

## Задача

Показать заказчику — проектировщику, инженеру, техническому заказчику — **где и как** работает оборудование ANHEL. Это именно то, о чём говорил владелец: «чтобы можно было реально видеть, как это применяется в жилом доме или промышленном здании».

## Две версии реализации

### Вариант A (рекомендуется): интерактивный 3D-разрез

Three.js + React Three Fiber. Пользователь видит стилизованный изометрический разрез здания с кликабельными маркерами установки оборудования.

### Вариант B (бюджетный): SVG scrollytelling

Крупная SVG-иллюстрация здания, при скролле камера движется снизу вверх, подсвечиваются узлы.

**Выбираем A, если бюджет и сроки позволяют.**

## Вариант A: 3D-разрез здания

### Технология

- **React Three Fiber** + **Drei** (помощники)
- Модели в формате **GLB** (Draco-сжатие)
- Размер моделей: ≤ 2 МБ каждая

### Сцена


### Типы зданий (переключатель сверху)

Каждый тип — отдельная 3D-сцена с соответствующим расположением оборудования:

1. **Жилой комплекс** (25 этажей)
   - Подвал: насосная водоснабжения + БИТП
   - Тех. этаж: водоподготовка
   - Периметр: система пожаротушения

2. **Бизнес-центр** (18 этажей)
   - Подвал: пожаротушение (бОльшая производительность)
   - Цоколь: тепловой пункт
   - Технический этаж: водоподготовка

3. **Завод / промышленное здание**
   - Технологический узел водоподготовки (приоритет)
   - Насосные станции для техпроцессов
   - Система пожаротушения высокой категории

4. **ТРЦ / общественное здание**
   - Пожаротушение (дренчерные системы)
   - Водоснабжение
   - Тепловой пункт для вентиляции

### Интерактивные маркеры (Pulse pins)

На 3D-модели размещены маркеры с pulse-анимацией:

```jsx
<mesh position={[x, y, z]} onClick={() => openPopup('water-supply')}>
  <sphereGeometry args={[0.05, 16, 16]} />
  <meshBasicMaterial color="#1E6FD9" />

  {/* Pulse-анимация вокруг */}
  <mesh>
    <sphereGeometry args={[0.1, 16, 16]} />
    <meshBasicMaterial color="#1E6FD9" transparent opacity={0.3} />
  </mesh>
</mesh>
```

### При клике на маркер

1. Камера плавно зумится к точке (1.2s, ease-in-out)
2. Справа выезжает панель с информацией:
   - Название оборудования
   - Роль в системе здания
   - Ключевые параметры
   - Ссылка на страницу продукта
3. Кнопка «Закрыть» возвращает камеру в общий вид

### Камера и управление

- **Начальный вид:** изометрия, 45° сверху
- **OrbitControls** (только вращение, zoom ограничен)
- **Auto-rotate** в idle (медленно, 30°/мин)
- **dampingFactor:** 0.05 (плавность)

```jsx
<OrbitControls
  enablePan={false}
  enableZoom={true}
  minDistance={10}
  maxDistance={30}
  maxPolarAngle={Math.PI / 2}
  autoRotate={idle}
  autoRotateSpeed={0.5}
  dampingFactor={0.05}
/>
```

### Переключение между типами зданий

Плавная морфинг-анимация 1.5s между моделями:

```javascript
// Framer Motion на 3D-уровне через spring
const { position, rotation, scale } = useSpring({
  from: oldBuilding,
  to: newBuilding,
  config: { duration: 1500, easing: 'easeInOut' },
});
```

### Производительность

- Загружать 3D-модели **лениво** (только при входе секции в viewport)
- Fallback: пока модели грузятся — показывать статичный SVG-превью с маркерами
- Обязательно: `Suspense` + loading state

```jsx
<Suspense fallback={<StaticBuildingPreview />}>
  <Canvas>
    <Building type={activeType} />
  </Canvas>
</Suspense>
```

## Вариант B: SVG scrollytelling (мобильный fallback + бюджетная версия)

### Механика

Крупная SVG-иллюстрация здания на весь экран. При скролле **секции** (pin через GSAP) камера движется снизу вверх по зданию, подсвечивая узлы оборудования.

### SVG-структура

```xml
<svg viewBox="0 0 800 1600">
  <!-- Здание -->
  <g id="building">
    <rect x="100" y="0" width="600" height="1600" fill="#1A1A1A" />
    <!-- этажи -->
    <line x1="100" y1="60" x2="700" y2="60" stroke="#333" />
    <!-- ... -->
  </g>

  <!-- Слой оборудования, изначально скрыт -->
  <g id="equipment" opacity="0">
    <g id="water-supply" transform="translate(400, 1500)">
      <rect width="80" height="40" fill="#1E6FD9" />
      <text>Водоснабжение</text>
    </g>
    <!-- ... -->
  </g>

  <!-- Выноски -->
  <g id="callouts" opacity="0">
    <!-- линии + подписи -->
  </g>
</svg>
```

### ScrollTrigger-анимация

```javascript
const tl = gsap.timeline({
  scrollTrigger: {
    trigger: '.applications-section',
    pin: true,
    scrub: 1,
    start: 'top top',
    end: '+=300%',
  },
});

// Шаг 1: камера снизу, подсвечивается подвал (водоснабжение)
tl.to('#camera', { y: -1200, duration: 1 })
  .to('#water-supply', { opacity: 1, scale: 1.1, duration: 0.3 }, '<')
  .to('#water-supply-callout', { opacity: 1, duration: 0.3 }, '<0.2')

  // Шаг 2: поднимаемся к БИТП
  .to('#camera', { y: -900, duration: 1 })
  .to('#water-supply', { opacity: 0.5, scale: 1 }, '<')
  .to('#heat', { opacity: 1, scale: 1.1 }, '<')

  // ... и т.д.
```

## Контент (тексты для маркеров)

### Жилой дом

**Подвал — Насосная станция водоснабжения (HVS-NU)**
> Поднимает воду до 25 этажа с постоянным давлением. Автоматический режим работы — 2 насоса в работе, 2 в резерве. Защита от «гидроудара».

**Подвал — Блочный индивидуальный тепловой пункт (БИТП)**
> Готовит горячую воду и отопление для всех 500 квартир. Автоматика поддерживает температуру независимо от погоды. Экономия тепла — до 30%.

**Периметр — Насосная станция пожаротушения**
> Запускается автоматически при срабатывании сигнализации. Создаёт давление для спринклеров на всех этажах за 90 секунд (по СП 10.13130).

**Тех. этаж — Установка водоподготовки**
> Очищает воду от жёсткости, железа, хлора перед подачей в квартиры. Бытовая техника жителей служит в 2-3 раза дольше.

*(Аналогично для остальных типов зданий)*
