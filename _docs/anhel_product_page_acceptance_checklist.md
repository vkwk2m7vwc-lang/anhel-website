- Если опросный лист реализован как интерактивная форма — работает ли он корректно

- Корректное ли имя у PDF-файла
- Есть ли возможность заполнить опросный лист с сайта
- Есть ли возможность отправить заполненный опросный лист с сайта
- Не пугает ли форма пользователя объёмом полей
- Есть ли пояснение, зачем нужен опросный лист
- Понятно ли, что можно заполнить его позже или отправить проектную документацию

- Не хвали без причины
- Главная задача — найти слабые места
- Цель — довести страницу до уровня премиального B2B-сайта
- Если секция не реализована — пометь явно «не реализовано»
- Если секция не реализована, не снижай оценку задним числом, но зафиксируй как gap
- Двухшапочная проблема секции 3 — известный баг
- Двухшапочную проблему отметить, но не включать в ТОП-10
- Не исправляй код в рамках аудита
- Не делай коммиты
- Не меняй файлы
- Только отчёт
- Фиксы будут отдельной итерацией после приоритизации

---

# ПОСЛЕ ОТЧЁТА

После отчёта:

1. Сформируй ТОП-10 правок
2. Не начинай реализацию без отдельной команды
3. После согласования правок подготовь план фиксов по итерациям
4. Только после отдельной команды переходи к исправлению
