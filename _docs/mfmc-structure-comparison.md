# Структурное сравнение секций — mfmc.ru vs anhel-website

Сравнение порядка и состава секций на исходных продуктовых страницах
(`mfmc.ru/catalog/...` и `anhelspb.com/catalog/...`) и нашей текущей
10-секционной структуры. Цель — показать какие секции есть у источников,
которых нет у нас, и обоснованно обсудить нужно ли добавлять.

## Наша текущая структура (10 секций)

Применяется ко всем 4 продуктовым страницам:

```
01  Hero
02  ТТХ
03  Применение / Линейка модулей       ← у heating-unit заменено на каталог 8 модулей
04  Бренды
05  Преимущества
06  Галерея
07  Кейсы
08  Опросный лист (квиз)
09  Документация
10  Финальный CTA
```

## Структура mfmc.ru (типовая страница продукта, например `nasosnye-ustanovki-alfa-dlya-sistem-vodosnabzheniya/`)

```
01  Hero (название + одна-две строки tagline)
02  ТТХ (6-9 строк, label/value)
03  Область применения (4-6 буллетов)
04  Преимущества наших насосных станций (8-10 строк)
05  Собираем на базе насосов: <бренды>
06  Используем комплектующие: <бренды>
07  Документация (декларация, сертификаты, руководство)
08  Реализованные проекты (1-2 кейса)
09  Из этой же категории (cross-link на соседние продукты)
10  «Где купить» / контакты компании
11  Расширенный текст (длинный SEO-абзац внизу страницы)
```

## Структура mfmc.ru/catalog/sigma/<module> (страница модуля БТП)

```
01  Hero (название + tagline)
02  ТТХ (5-8 строк)
03  «Область применения» / «Назначение» (буллеты)
04  Документация (опросный лист + декларация)
05  Реализованные проекты
06  Из этой же категории (соседние модули)
07  Контакты / форма быстрого запроса
08  «Развёрнутый текст» — длинный SEO-блок (функциональные задачи, принцип работы)
```

## Структура anhelspb.com/catalog/waterpreparation/

```
01  Hero (краткое описание)
02  Область применения (4 пункта)
03  Документация (декларация)
```

Лаконично — нет ТТХ, нет преимуществ, нет кейсов, нет брендов
комплектующих. По объёму — это ~10% типовой страницы mfmc.ru.

## Что ЕСТЬ у mfmc.ru, ЧЕГО НЕТ у нас

### 1. Развёрнутый SEO-текст внизу страницы

На mfmc.ru каждая продуктовая страница после стандартных блоков имеет
длинный (1500-3000 знаков) текстовый раздел с подзаголовками
«Сфера применения и преимущества», «Где купить», «Функциональные
задачи модуля». Цель — SEO-оптимизация под коммерческие запросы.

**Моё мнение**: добавить ОТДЕЛЬНУЮ секцию `SeoLongRead` или
`AboutTheProduct` в наш template. Аудитория — поисковики и пользователи
которые «прокрутили всё, хотят больше». На mobile секция сворачивается
в accordion («Подробнее о продукте»). Не критично для UX (B2B-
читатель уже скачивает опросный лист и заполняет квиз), но важно
для SEO-видимости.

**Приоритет**: M. Добавить в Stage 7 после получения реальных кейсов
и фото — иначе SEO-текст без поддерживающих визуалов смотрится «пустым».

### 2. «Особенности эксплуатации» / «Принцип работы»

На некоторых страницах mfmc.ru (особенно heating-unit) есть текстовое
описание принципа работы — например для `modul-vvoda-i-ucheta-tepla/`
там полный абзац «Узел учёта теплопотребления выполняется в
соответствии с Постановлением Правительства РФ №1034...».

Сейчас у нас этот контент частично перенесён в `description` поле
heating-modules data.ts. Но на главных страницах (firefighting,
water-supply, water-treatment) аналога нет.

**Моё мнение**: heating-unit подстраницы уже покрывают это через
секцию «Описание». Для других продуктов это пока **не** требуется —
там другая природа (насосная станция работает по очевидной механике,
описывать её принцип = воду в ступу). Для heating-unit — оставить
как есть.

**Приоритет**: L. Не добавлять.

### 3. Развёрнутая «Сфера применения» (с описанием каждого случая)

На mfmc.ru применение часто оформлено как длинные параграфы по
2-4 предложения на тип объекта («Спортивные комплексы, медицинские
учреждения и выставочные центры также нуждаются в надёжных системах
подачи воды. Насосные установки АЛЬФА Stream® способны
удовлетворить требования таких объектов благодаря адаптивности.»).

У нас «Применение» — короткий title + example в одну строку. Это
визуально аккуратно для гридов, но беднее на смыслы.

**Моё мнение**: можно расширить current ApplicationItem типом, добавив
optional `body` поле (1-2 предложения). На mobile показывать только
title+example (как сейчас); на desktop — раскрывать body на hover или
в отдельном блоке. Это улучшит SEO и B2B-аргументацию.

**Приоритет**: M. Можно сделать в Stage 7.

### 4. «Из этой же категории» как explicit cross-link

На mfmc.ru это отдельная секция с 3 карточками соседних продуктов из
той же категории (например, на странице водоснабжения — water-supply,
firefighting, поддержание давления).

У нас в `ProductCtaFooter` уже есть `neighboursCaption: "Остальные продукты"`
— но он показывает 4 ВСЕХ продукта (3 кроме текущего). На mfmc.ru
делается фокус на 2-3 «ближайших по смыслу» (категория = насосы или
категория = тепло).

**Моё мнение**: текущий вариант ОК для маленького каталога (4 продукта).
Если категорий станет больше — рефакторить.

**Приоритет**: L.

### 5. «Где купить» (контактный блок в нижней части продуктовой страницы)

На mfmc.ru в самом низу страницы — повтор контактов, адрес производства,
ссылки на сертификаты. У нас аналог рассыпан между hero (с быстрым
запросом), Footer (с контактами компании), и `ProductCtaFooter`
(c CTA на квиз).

**Моё мнение**: текущая раскидка работает хорошо. Footer виден на
каждой странице, hero CTA сверху — этого достаточно. Дублировать
контакты в продукт-теле — лишний визуальный шум.

**Приоритет**: L. Не добавлять.

## Что есть у нас, чего НЕТ у mfmc.ru/anhelspb.com

### 1. Опросный лист как 6-шаговый интерактивный квиз

На mfmc.ru опросный лист — это PDF, который надо скачать и заполнить.
У нас — интерактивная форма с валидацией (email, phone), которая
отправляется в админку.

**Conviction**: оставить, это ключевая UX-конверсия для B2B.

### 2. Галерея производства / монтажа

У нас — секция `GalleryRail` с 8 photo-skeletons (заполнится после
photoshoot). На mfmc.ru галерея отсутствует — есть только один
hero-render продукта.

**Conviction**: оставить, это существенный premium-фактор.

### 3. Hero с большим 3D-рендером + accent-glow + tilt

У нас — hero занимает 88svh с продуктом справа в 45% колонки, accent-tinted
ambient + pedestal-glow + magnetic CTA. На mfmc.ru hero — стандартная
H1 + tagline + одна статичная картинка над текстом.

**Conviction**: оставить, это часть premium-визуала и referencing
terminal-industries.com / mont-fort.com.

### 4. Section 3 firefighting — interactive Lakhta scene

Уникальная секция, которой нет у mfmc.ru. SVG-рендер «Лахта Центра» с
6 интерактивными шагами + 4 системы пожаротушения (sprinkler / drencher /
ВПВ / combined).

**Conviction**: ключевой differentiator, оставить.

### 5. Каталог 8 модулей heating-unit с dynamic-подстраницами

У mfmc.ru тоже 8 модулей в /catalog/sigma/, но каждый — отдельная
страница в общем каталоге. У нас они все живут под /products/pumps/
heating-unit/[slug] с единым tone и navigation back-to-catalog.
UX-эквивалент, но более каталогизированный.

**Conviction**: оставить.

## Итог

**Что точно стоит добавить (приоритет M)**:
1. **Развёрнутый SEO-текст** внизу каждой продуктовой страницы (после
   ProductCtaFooter, в виде свернутого `AboutTheProduct` accordion на mobile).
2. **`body` поле в `ApplicationItem`** для более содержательной
   «Применение» секции (опционально на desktop).

**Что не нужно добавлять**:
1. «Принцип работы» как отдельная секция — только для heating-unit
   подстраниц, уже есть в description.
2. «Где купить» — текущий Footer достаточен.
3. «Из этой же категории» — текущий ProductCtaFooter neighbours
   уже работает.

**Что у нас уже сильнее, чем у источников**:
1. 6-шаговый интерактивный квиз вместо PDF.
2. Галерея производства.
3. Hero с 3D-рендером + tilt.
4. Section 3 firefighting с Lakhta scene.
5. Каталог 8 модулей heating-unit с dynamic routes.
